# React 组件生命周期

React 组件生命周期 http://react.caibaojian.com.cn/docs/react-component.html (opens new window)

每个组件都有几个“生命周期方法”，可以重写这些方法，已在过程中的特定时间运行代码。组件的声明周期大概分为三个阶段：

1. Mounting(装载)
2. Updating(更新)
3. Unmounting(卸载)

# 阶段一、Mounting(装载)

当组件实例被创建并将其插入 DOM 时，将按以下顺序调用这些方法

  • # constructor() 构造函数，两个目的:

    • 通过分配一个对象到 this.state 来初始化本地state，如：this.state = { counter: 0 };
    • 将事件处理程序方法绑定到实例，如：this.handleClick = this.handleClick.bind(this);

  • # componentWillMount() UNSAFE_

    在数据挂载之前立即执行

  • # render()

    render() 方法是类组件中唯一必须的方法；

  • # componentDidMount()

    在数据挂载后立即执行，初始化所需要的DOM节点、远程加载数据、网络请求应该放这里。

    可以在 componentDidMount() 中调用 setState(), 在浏览器更新屏幕之前，会触发额外的渲染，在这种情况下，即使 render()会被调用两次，也可以保证用户不会看到中间状态。谨慎使用，通常会导致性能问题

# 阶段二、Updating(更新)

当 props 或 state 发生变更时会引起组件重新渲染，按以下顺序调用这些方法

  • # componentWillReceiveProps(nextProps)

在以装载组件接收 新props 之前被调用。如果需要更新 state 以响应 props的更改，可以在此方法中比较 this.props 和 nextProps并使用this.state执行状态转换。

注意

即使 props 没有改变，React也可能调用这个方法，如果只想处理变化，确保比较当前值和下一个值。当父组件导致子组件重新渲染时，可能发生这种情况

  • # shouldComponentUpdate(nextProps, nextState)

当接收到新的属性(props) 或 状态(state) 时，shouldComponentUpdate()在渲染之前被调用。默认返回 true。

此方法仅作为 性能优化 存在，不要依靠它来防止渲染。可以使用内置的 PureComponent，而不是手动编写此方法，PureComponent 对props 和 state 进行浅层比较。

注意

  • 如果 shouldComponentupdate()返回false,那么UNSAFE_componentWillUpdate()、render()、componentDidUpdate()不会被调用。
  • 不推荐进行深度相等检查或者在 shouldComponentUpdate()中使用JSON.stringify(),非常低效，会降低性能。

  • # componentWillUpdate()

当接收到新的属性(props) 或 状态(state) componentWillUpdate()在渲染之前被调用。第一次渲染不会被调用。

注意

这里不能调用 this.setState()，不应该做任何其他事。
通常使用 componentDidUpdate()代替。

  • # render()

  • # componentDidUpdate(prevProps, prevState, snapshot)

componentDidUpdate()在更新发生后立即被调用，第一次渲染时不会被调用。

当组件已更新时，可以在此操作DOM，也可以发起 网络请求，只需你比较当前的 props 与 以前的 props。

# 阶段三、Unmounting(卸载)

当一个组件从 DOM 中删除时，将调用此方法：

  • # componentWillUnmount()

在一个组件被 卸载(unmounted) 和 销毁(destroyed)之前立即被调用。在此方法中执行必要的清理，如：清除定时器，取消网络取消等等，

例子：在 componentDidMount() 中添加一个全局点击事件：

document.addEventlistener('click', this.closeMenu)
1

添加closeMenu() 方法：

closeMenu(){
    console.log('-------------- 点击 closeMenu方法')
}
1
2
3

在 componentDidUnmount() 方法中清除点击事件，如果不清除，组件卸载时点击事件依然存在

document.removeEventListener('click', this.closeMenu)
1

注意

不应该在 componentWillUnmount()中调用setState(), 因为组件不会被重新渲染。

# 错误处理

在生命周期方法中，或在任何子组件的构造函数中，渲染过程中出现错误时调用此方法。

  • # componentDidCatch(error, info)

可以在其子组件树中的任何位置捕获JavaScript 错误，记录这些错误，并显示备用 UI 而不是崩溃的组件树
