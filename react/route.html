# React路由

后端路由缺点：每次切换路由的时候都需要刷新页面，然后发送 Ajax 请求，将请求数据返回，这样体验很不好，因此出现了前端路由。

# 路由原理

SPA: Single Page Application 单页应用

例如：https://music.163.com/#/ (opens new window) 路由用来管理 url地址 和 视图 之间的关系

[路由原理参考]https://blog.csdn.net/qq_30012951/article/details/108703476 (opens new window)

  • 准备视图(html页面)
  • 准备路由的路线(路由与视图的对应关系， 可以是一个对象，键名是路线名称， 值是视图地址)
  • 通过hash地址的路线，获取对应的视图地址
  • 在指定标签中加载路由对应的视图

代码演示：

router.html代码如下：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>路由原理</title>
    <script src="https://cdn.bootcss.com/jquery/1.12.4/jquery.min.js"></script>
</head>
<body>
    <ul>
        <!-- 
            路由 location.hash
            1、href="./index.html" 会跳转到了另外一个页面
            2、href="#/list" 单页面应用 SPA(路由中使用#/XXXX 跳转) 在当前页面加载另外一个页面的内容 
         -->
        <li><a href="#/index">首页</a></li>
        <li><a href="#/list">列表</a></li>
        <hr>
        <!-- 视图展示区域 -->
    </ul>
    <div id="routerView"></div>

    <script>
        // 定义一个路由（多个路线， 多个对应关系）
        let routes = {
            // 描述url 和 视图 的对应关系
            '#/index': './index.html',
            '#/list': './list.html',
        }
        
        // 给窗口添加事件，当哈希值（路由）发生变化的时候， #routerView盒子加载对应的视图
        window.addEventListener('hashchange', () => {
            // 查看当前 url 路由
            console.log('路由改变: ',location.hash);
            // 加载路由对应的视图
            $('#routerView').load(routes[location.hash])
        })

    </script>
</body>
</html>

# Hash 模式

hash 路由模式是这样的：http://127.0.0.1:5500/src/routerHtml/router.html#/list。有带 # 号，后面就是 hash 值的变化。 改变后面的 hash 值，不会刷新浏览器。并且每次 hash 值改变的时候， 会触发 hashchange 事件，因此只需监听此事件就可知道hash值

// 给窗口添加事件，当哈希值（路由）发生变化的时候， #routerView盒子加载对应的视图
window.addEventListener('hashchange', () => {
    // 查看当前 url 路由
    console.log('路由改变: ',location.hash);
    // 加载路由对应的视图
    $('#routerView').load(routes[location.hash])
})

URL地址详解：

属性 描述
location.href 完整的url
location.protocal 当前URL的协议，包括https等
location.host 主机名和端口号 如：www.baidu.com:8080
location.hostname 主机名 如： www.baidu.com
location.port 端口号；比如： 8080
location.pathname url的路径部分，从 / 开始，比如 https://www.baidu.com/s?ie=utf-8，那么pathname = '/s'
location.search 查询参数，从？开始， 比如 https://www.baidu.com/s?ie=utf-8 那么 search = '?ie=utf-8'
location.hash hash是URL中的一个片段，从 # 开始，比如：https://www.baidu.com/#/a/b ，那么hash就是 '#/a/b'

# History模式

HTML5 的History API 为浏览器的全局history对象增加了该扩展的方法。它是浏览器的一个接口，在 window对象中提供了 onpopstate 事件来监听历史栈的变化。只要历史栈有信息发生改变，就会触发该事件。

# React路由基本使用

  • 可以实现页面的局部刷新
  • 按需加载子组件
  • 子组件可以使用相对路径，继承父路线来作为前缀如：父路线 path="/web", 子路线 path="index" 访问的路径就是： XXX/web/index

安装React路由

npm install react-router

在路由配置模块中引入路由：

import { Router, Route, hashHistory} from "react-router"

配置路由器(路由与视图的关系):

const router = <Router>
        <Route path="路由1" component={视图组件1}></Route>
        <Route path="路由2" component={视图组件2}></Route>
        <Route  path="嵌套路由1" component={嵌套视图组件1}>
            <Route path="" component={} ></Route>
        </Route>
    </Router>

在 App.js(父组件) 中引入相关组件，配置路由，渲染路由，在父组件中使用{ this.props.children } 来获取子路线组件视图（必须当子组件激活时，才会去创建对应的子组件，并且挂载到 this.props.children ）

// 引入核心包
import React, {Compoenent} from "react"
import ReactDOM from "react-dom"

// 引入视图组件
import Index from "./index"
import List from "./list"

// 引入React路由 react-router
import { Router, Route, hashHistory} from "react-touter"

export default class APP extends Compoenent{
    render(){
        return (
            <div>
                <ul>
                    {/* <li><a href="#/app/index">首页</a></li>
                    <li><a href="#/app/list">列表页</a></li> */}
                    <li><a href="#/index">首页</a></li>
                    <li><a href="#/list">列表页</a></li>
                </ul>
                <hr />
                {/* this.props.children 获取当前激活的子组件（子路由） */}
                {this.props.children}
            </div>
        )
    }
}

// 定义路由器 
const router = <Router>
    {/* 这种写法会跳转到另一个页面 */}
    {/* <Route path="/index" component={Index}></Route> 
    <Route path="/list" component={List}></Route>  */}
    {/* 单页应用 跳转 嵌套路由 path中不需要写 # */}
    <Route path="app" component={App}>
        {/* 相对路由 */}
        <Route path="index" component={Index}></Route>
        {/* 绝对路由 */}
        <Route path="/app/list" component={List}></Route>
    </Route>
</Router>

ReactDOM.render(router, document.getElementById('root'))

// 在入口文件引入 App.js 文件即可

# React路由参数

定义路由时，在 path 上声明参数规则，如：

<Route path="/app/detail/:newsId" component={Detail}></Route>
1

使用 detail/ 后面的 : 来动态匹配参数，newsId 为自定义传到要跳转的路由 视图的参数名；

使用连接地址的时候传参：

< href="#/app/detail/123"></a>
1

123 就是要传的参数值

在路线对应的视图组件中，使用this.props.routeParams.变量名 获取

this.props.routeParams.newId // 获取参数 newsId 的值 在componentDidMount() 中发起请求
1

在Detail.js 中：

import React, { Component } from 'react'

export default class Detail extends Component {

    componentDidMount(){
        // 发起 Ajax 请求
        console.log(this.props.routeParams.newsId)
    }

    render() {
        return (
        <div>
            <h1>详情页：通过 this.props.routeParams.参数名 接收参数</h1> 
            参数：{ this.props.routeParams.newsId}
        </div>
        )
    }
}

# React其他路由组件

  • IndexRoute
    默认路由：用于设置默认展示的页面
  • IndexRedirect
    重定向路由
  • Link
    a 标签的封装

使用：

引入组件：

import { IndexRoute, Link, IndexRedirect} from react-router
1

在路由器嵌套路由中使用，如：

// 在上述代码中使用
// IndexRoute 在保持 /app 路径不变的情况下， 设置默认展示页面 访问 /app 是展示首页
<IndexRoute component={Index}></IndexRoute>

// IndexRedirect 在访问 /app 时 重定向到 /app/index 页面 
<IndexRedirect to="index"></IndexRedirect>

// Link 的使用  底层由 a 标签
<li><Link to="/app/index">首页</Link></li>
<li><Link to="/app/list">列表页</Link></li>
<li><Link to="/app/detail/123">详情页</Link></li>
